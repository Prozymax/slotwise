# Demo script — 3 minutes, numbers verified by `npm run verify-demo`

> Pin this dataset. Rehearse these EXACT drags in this EXACT order. 5+ run-throughs.

**[0:00 — The problem]**
"Every multi-track conference agenda is built the same way: a spreadsheet, a program chair,
and a prayer. Meanwhile registration already captured what every attendee wants to see —
and that data is never used. The result is the #1 complaint at every conference: two talks
you love, same time slot — and a 350-person crowd outside an 80-seat room."

**[0:25 — The reveal]**
Open Slotwise. "This is DevConf Lagos — 20 sessions, 3 rooms, and 1,200 real registrations
imported. The spreadsheet agenda scores like this:" → point at header:
**729 attendees impacted · 363 with clashes · 366 seats short · 3 rooms over capacity.**
Point at the worst-clash banner: "152 people want both of these talks. They're head-to-head."

**[0:50 — DRAG 1]**
"Fine-Tuning on a Shoestring is clashing with our most popular talk AND overflowing Room Beta.
Watch the score." → Drag **s02 "Fine-Tuning on a Shoestring" → Main Hall @ 10:00** (swaps with
Prompt Injection). Header: **653 → impacted; clashes 363→287, seats short 366→293.** "One drag.
Seventy-six people just got their schedule back."

**[1:20 — DRAG 2]**
"Our single most popular session — 289 people expected — is in the smallest room in the
building." → Drag **s01 "LLMs in Production" → Main Hall @ 9:00**. Header: **clashes 287→248,
seats short 293→114, overflows 2→1.** Flip to the **Room load** tab — bars dropping out of the
red band — flip back.

**[1:50 — DRAG 3]**
"One overflow left." → Drag **s06 "React Server Components" → Main Hall @ 13:00**.
Header goes green: **186 impacted · 0 seats short · 0 rooms over capacity.**
"Zero overflows. The remaining 186 are cross-interest clashes — and now the organiser can
*see* them instead of discovering them in the post-event survey."

**[2:20 — The close]**
"Same sessions. Same speakers. Same rooms. Three drags, and 543 more people get the
conference they registered for. Slotwise closes the broken pipe between registration data
and the scheduling decision — built on Kendo's Scheduler, Grid, and Charts. Spreadsheets
display agendas. Slotwise optimises them."
